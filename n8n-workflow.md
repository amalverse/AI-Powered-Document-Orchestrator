# N8N AI Document Orchestrator Example Workflow

This is an example n8n workflow file for an AI-powered document orchestrator that analyzes documents using Groq AI and optionally sends email alerts.

## Overview

This workflow demonstrates how to:
- Receive document analysis requests via webhook
- Process documents using Groq AI (Llama 3.3 model)
- Generate structured JSON responses with analysis and email content
- Conditionally send email alerts based on user input

## Prerequisites

- n8n instance (self-hosted or cloud)
- Groq API key (sign up at https://groq.com)
- Gmail account with OAuth2 credentials (for email functionality)

## Setup Instructions

### 1. Import the Workflow

1. Download the `n8n-example-workflow.json` file
2. In your n8n dashboard, click "Import" or "New Workflow" > "Import from File"
3. Select the downloaded JSON file

Note: This workflow uses placeholder credentials. You will need to configure your own credentials for the Groq API and Gmail.

Note: You can create n8n workflows from scratch from the n8n editor, or import this workflow from the provided JSON file. If you choose to create it from scratch, follow the node configuration steps below.

### 2. Configure Credentials

#### Groq AI API
1. Go to your n8n credentials section
2. Create a new credential for "HTTP Request" (or use existing)
3. Set the Authorization header to: `Bearer YOUR_GROQ_API_KEY`
4. Replace `YOUR_GROQ_API_KEY` with your actual Groq API key

#### Gmail (Optional)
1. Create Gmail OAuth2 credentials in n8n
2. Configure the "Send Gmail Address" node with your credentials
3. Ensure your Gmail account allows less secure app access or use OAuth2

### 3. Activate the Workflow

1. Review all nodes and ensure configurations are correct
2. Click "Activate" to enable the webhook endpoint
3. Note the webhook URL provided by the Webhook node

## Usage

### API Endpoint

The workflow exposes a POST endpoint at `/webhook/document-orchestrator`

### Request Format

Send a POST request with JSON body containing:

```json
{
  "query": "Your analysis question here",
  "text": "The document text to analyze (up to 20,000 characters)",
  "email": "optional@email.com"  // Optional: email address for alerts
}
```

### Response Format

The workflow returns a JSON response:

```json
{
  "answer": "Detailed analysis of the document based on your query",
  "email_body": "<html>Formatted email content</html>",
  "status": "Email Sent Successfully" | "Warning: No email provided, email was not sent."
}
```

## Workflow Nodes Explanation

1. **Webhook**: Receives POST requests with document data
2. **Groq AI - Answer & Email**: Sends document and query to Groq AI for analysis
3. **IF Email Exists**: Checks if an email address was provided
4. **Send Gmail Address**: Sends formatted email alert (if email provided)
5. **Respond to Webhook**: Returns JSON response with analysis results

## Customization

### AI Model
- Currently uses `llama-3.3-70b-versatile`
- Modify the model parameter in the Groq AI node for different models

### Email Template
- Email content is generated by the AI based on the analysis
- Customize the prompt in the Groq AI node to change email formatting

### Response Structure
- The AI is instructed to return JSON with `answer` and `email_body` keys
- Modify the prompt to change the response format

## Security Notes

- This example uses placeholder credentials
- Never commit real API keys or credentials to version control
- Use n8n's credential management for secure key storage
- Consider rate limiting and authentication for production use

## Troubleshooting

### Common Issues

1. **Webhook not receiving requests**: Ensure the workflow is activated
2. **AI analysis fails**: Check your Groq API key and quota
3. **Email not sending**: Verify Gmail credentials and OAuth2 setup
4. **JSON parsing errors**: Ensure the AI response format matches expectations

### Logs
Check n8n's execution logs for detailed error messages and debugging information.
